Event Validation

Validation rejects a callback whose event time is older than the threshold below, so stale/queued callbacks never fire a late notification. This is separate from deduplication.

e.g. 60 = ignore a callback whose event time is over a minute ago.

Deduplication

Deduplication collapses the burst of callbacks one real interaction emits (e.g. a ring + ring-to-open) so you get a single notification. Different from validation: validation drops old events, deduplication drops repeated ones within the window.

e.g. 120 = collapse repeated callbacks for the same interaction seen within two minutes.
